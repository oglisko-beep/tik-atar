# תיק אתר — מערכות מידע (Organizational IT Portfolio)

תבנית-אב (master template) לתיעוד מערכות המידע של אתר/בניין בארגון.
התבנית גנרית — משכפלים אותה וממלאים מחדש לכל אתר.

> 🆕 **גרסה אינטראקטיבית (2026):** קיימת כעת אפליקציית web למילוי התיק בדפדפן —
> טפסים, שמירה אוטומטית, ריבוי אתרים, חיפוש, מצב כהה, הדפסה/PDF וייצוא Word.
> ראו תיקיית [`app/`](app/README.md). הרצה מהירה: `cd app && npm install && npm run dev`.

## תוכן הפרק (המסמך)
שער + לוגו גב-ים · תוכן עניינים · בקרת מסמך · פרטי אתר ·
1. ציוד קצה · 2. בקרות כניסה · 3. רשת ושרתים · 4. תוכנות ורישוי ·
5. אבטחת מידע · 6. הגנת סייבר · 7. ספקים · 8. אנשי קשר · 9. נספחים.

## קבצים בתיקייה
| קובץ | תיאור |
|------|-------|
| `תיק אתר - מערכות מידע (תבנית-אב).docx` | **המסמך עצמו** — התוצר המוכן לשימוש |
| `generate.js` | סקריפט שמייצר את המסמך (מקור האמת) |
| `gav-yam.svg` | לוגו גב-ים (מוטמע אוטומטית בשער) |
| `node_modules/`, `package.json` | תלויות (docx + ממיר SVG) |
| `app/` | 🆕 **האפליקציה האינטראקטיבית** (Vite + React + TS) — ראו `app/README.md` |
| `docs/superpowers/` | מסמכי עיצוב ותכנון של האפליקציה |

## איך מחדשים / משנים את המסמך
1. עורכים את התוכן ב-`generate.js` (פרקים, טבלאות, שדות).
2. מריצים מהתיקייה:
   ```powershell
   node generate.js
   ```
   הפלט נכתב אוטומטית כ-`תיק אתר - מערכות מידע (תבנית-אב).docx` באותה תיקייה.
3. אפשר גם פשוט לפתוח את ה-`.docx` ב-Word ולמלא ידנית (אז אין צורך ב-generate.js).

> אם `node_modules` נמחקת אי-פעם, התקנה מחדש:
> ```powershell
> npm install docx @resvg/resvg-js @resvg/resvg-js-win32-x64-msvc
> ```
> (חובה את החבילה האחרונה במפורש — אחרת הלוגו מדולג בשקט.)
