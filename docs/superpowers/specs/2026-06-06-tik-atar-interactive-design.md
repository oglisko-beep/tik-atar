# תיק אתר אינטראקטיבי — מערכות מידע · מסמך עיצוב (Spec)

> תאריך: 2026-06-06 · סטטוס: מאושר עקרונית, ממתין לתוכנית מימוש
> מבוסס על: `generate.js` (תבנית-אב ל-Word) — מקור האמת לתוכן

## 1. רקע ומטרה
היום קיים `generate.js` שמייצר מסמך Word סטטי ("תיק אתר — מערכות מידע, תבנית-אב") עבור גב-ים.
המטרה: **אפליקציית web אינטראקטיבית מודרנית (2026)** שמחליפה את חוויית ה-Word —
מילוי טפסים בדפדפן, שמירה, ניווט, חיפוש, מחווני השלמה, הדפסה/PDF וייצוא, תוך נאמנות מלאה
לתוכן ולמותג הקיימים.

המסמך הוא **תבנית-אב** שממלאים מחדש לכל אתר — ולכן האפליקציה תתמוך ב**ריבוי אתרים**.

## 2. משתמשים ותרחישי שימוש
- **אחראי IT / מנהל מערכות מידע** ממלא ומתחזק תיק לכל אתר.
- **מנהל אבטחת מידע / סייבר** ממלא את פרקי הבקרות (5, 6).
- **מנהל אתר / הנהלה** צופה, מאשר ומדפיס.

תרחישים: יצירת תיק לאתר חדש · מילוי הדרגתי עם שמירה אוטומטית · חיפוש שדה ·
מעקב אחר אחוז השלמה · הדפסה/ייצוא PDF להפצה · גיבוי/שחזור JSON · ייצוא `.docx`.

## 3. עקרונות
- **נאמנות לתוכן**: כל פרק, תת-פרק, טבלה ושדה מ-`generate.js` מועתקים 1:1.
- **נאמנות למותג**: כחול `#104E7D`, צהוב `#EEC21B`, לוגו `gav-yam.svg`, עברית RTL מלאה.
- **schema-driven**: כל המבנה הוא נתונים; מנוע גנרי מרנדר ממנו. שינוי תוכן = עריכת סכמה.
- **אבטחה**: אין שמירת סיסמאות/מפתחות; סיווג "לשימוש פנימי"; הכל client-side (ללא שרת).
- **YAGNI**: בלי תכונות מיותרות; כל יכולת משרתת תרחיש אמיתי.

## 4. Stack ותלויות
- **Vite + React + TypeScript** (build step; `npm run dev` / `npm run build`).
- **CSS וניל + CSS Variables** ל-theming (כהה/בהיר), RTL וגיליון הדפסה. ללא Tailwind.
- **ללא שרת / ללא רשת בזמן ריצה** — נתונים ב-`localStorage` + ייבוא/ייצוא קבצים.
- תלות לייצוא docx (שלב אחרון): שימוש חוזר ב-`docx` (כבר בפרויקט) דרך bundler.
- אייקונים: סט SVG פנימי קל (ללא ספריית אייקונים כבדה).

## 5. ארכיטקטורה (schema-driven)

### 5.1 שכבות
- `app/src/schema/` — **הסכמה** (תוכן התיק כ-data). מקור האמת ב-runtime.
- `app/src/engine/` — **מנוע רינדור** גנרי: ממיר סכמה+ערכים ל-UI טפסים.
- `app/src/store/` — **state + persistence**: מצב, autosave, ייצוא/ייבוא, ריבוי אתרים.
- `app/src/ui/` — **shell**: ניווט, header, theme, חיפוש, מחווני השלמה.
- `app/src/print/` — תצוגת הדפסה/PDF + (שלב אחרון) ייצוא docx.

### 5.2 מודל הסכמה (types)
```ts
type Doc = { sections: Section[] }
type Section = { id: string; title: string; note?: string; blocks: Block[] }

type Block =
  | { kind: 'bullets'; title?: string; items: string[] }          // רשימות "מה לכלול"
  | { kind: 'kv'; id: string; fields: Field[] }                   // key-value (kvTable)
  | { kind: 'table'; id: string; columns: Column[];               // טבלה חוזרת (dataTable)
      examples?: Row[]; seedRows?: Row[]; minRows?: number; optional?: boolean }
  | { kind: 'checklist'; id: string; columns: Column[]; rows: string[] } // בקרות 5.1 / 6.1
  | { kind: 'box'; lines: string[] }                              // מציין-מקום (תרשים רשת)
  | { kind: 'callout'; tone: 'info'|'warn'; title: string; items: string[] }

type Field  = { id: string; label: string; type: FieldType; placeholder?: string; options?: string[] }
type Column = { id: string; label: string; type: FieldType; weight?: number; placeholder?: string }
type FieldType = 'text'|'textarea'|'date'|'select'|'ip'|'email'|'status'
type Row = Record<string,string>  // columnId -> value
```
- `status` = בחירה מתוך {קיים, חלקי, חסר, לא רלוונטי} עם תג צבעוני.
- `examples` = שורות ההדגמה האפורות מה-Word; מוצגות/מוסתרות במתג "הצג דוגמאות" (לא נספרות כהשלמה).
- `placeholder` נגזר מדוגמאות ה-`examples` בקוד המקורי.

### 5.3 מודל ה-state והשמירה
```ts
type SiteData = {
  id: string; createdAt: string; updatedAt: string;
  meta: { name: string; code: string; version: string; classification: string };
  values: Record<string /*blockId*/, KvValues | Row[]>;  // לפי סוג הבלוק
}
type AppState = {
  sites: Record<string, SiteData>;
  activeSiteId: string | null;
  ui: { theme: 'light'|'dark'; showExamples: boolean };
}
```
- שמירה ל-`localStorage` תחת מפתח מגורסן (`tik-atar/v1`), **debounced** (~500ms), עם מחוון "נשמר".
- ייצוא/ייבוא: קובץ JSON (אתר בודד או הכל). ייבוא מאמת גרסת-סכמה.
- ריבוי אתרים: `sites` ממופה לפי id; בורר אתרים ב-header; פעולות חדש/שכפל/מחק/שנה-שם.

## 6. מבנה התוכן (ממופה מ-`generate.js`, 1:1)
סדר הפרקים והבלוקים — זהה למסמך:

1. **בקרת מסמך** — note · טבלת היסטוריית גרסאות (גרסה/תאריך/עורך/תיאור/אישר) · טבלת אישורים (תפקיד/שם/חתימה/תאריך).
2. **פרטי אתר כלליים** — kv (12 שדות: שם, קוד אתר, כתובת, בניין/קומות, עמדות, משתמשים, שעות, אחראי IT, טלפון, דוא"ל, ספק תמיכה, תאריך איכלוס).
3. **1. ציוד קצה** — bullets · טבלה (7 עמודות).
4. **2. בקרות כניסה** — bullets · טבלה · CCTV (אופציונלי) · אזעקה kv (אופציונלי).
5. **3. רשת ושרתים** — 3.1 WAN · 3.2 ציוד תקשורת (bullets+טבלה) · 3.3 שרתים · 3.4 וירטואליזציה/אחסון · 3.5 IP/VLAN · 3.6 חדר תקשורת kv · 3.7 תרשים רשת (box).
6. **4. תוכנות ורישוי** — bullets · טבלה (7 עמודות).
7. **5. אבטחת מידע** — 5.1 checklist (14 בקרות) · 5.2 גיבוי kv (9) · 5.3 הרשאות טבלה · 5.4 תגובה לאירוע kv (6).
8. **6. הגנת סייבר** — 6.1 checklist (12 בקרות) · 6.2 SOC/SIEM kv (7) · 6.3 תגובה ודיווח kv (7) · 6.4 בדיקות חוסן טבלה · 6.5 ממשל kv (7).
9. **7. ספקים** — טבלה (6) · **8. אנשי קשר** — טבלה (6) · **9. נספחים** — bullets · callout אבטחה (אדום).

> מקור מדויק לכל עמודה/שדה/דוגמה: `generate.js`. הסכמה תועתק ממנו מילה-במילה.

## 7. רכיבי UI ומסכים
- **AppShell**: פריסת RTL — Sidebar (ימין) + Header (עליון) + Content.
- **Sidebar**: רשימת פרקים, לכל פרק טבעת/בר השלמה זעיר; הדגשת הפרק הפעיל; קיבוץ תת-פרקים.
- **Header**: בורר אתר · שם האתר · גרסה · תג סיווג · חיפוש (Cmd/Ctrl+K) · מתג כהה/בהיר · מחוון שמירה · תפריט (ייצוא/ייבוא/הדפסה/docx).
- **SectionView**: כותרת + note + רינדור הבלוקים.
- **רכיבי מנוע**: `KvBlock` · `TableBlock` (הוסף/מחק/שכפל שורה) · `ChecklistBlock` (בקרה + סטטוס צבעוני + כלי + אחראי) · `BulletsBlock` · `BoxBlock` (מציין-מקום) · `CalloutBlock`.
- **CommandPalette**: חיפוש שדות/פרקים וקפיצה אליהם.
- **CompletionMeter**: אחוז השלמה כולל + per-פרק.
- **PrintView**: תצוגת A4 ממותגת לכל הפרקים.

## 8. יכולות
- שמירה אוטומטית + מחוון מצב.
- ייצוא/ייבוא JSON (גיבוי/שיתוף).
- ריבוי אתרים (חדש/שכפל/מחק/שנה-שם/בחר).
- חיפוש מהיר (פקודה) על שמות שדות/פרקים/ערכים.
- מתג "הצג דוגמאות" (שורות הדגמה אפורות).
- מחווני השלמה (כולל + פר-פרק), עם התעלמות משדות אופציונליים מסומנים.
- ולידציות עדינות: IP, דוא"ל, תאריך (אזהרה לא-חוסמת).
- הערת אבטחה בולטת (אין סיסמאות).
- הדפסה/PDF ממותג.
- נגישות (ניווט מקלדת, ARIA, ניגודיות) + רספונסיבי (דסקטופ→טאבלט).

## 9. עיצוב ומותג
- צבעים: primary `#104E7D`, primaryDark `#0B3A5E`, accent `#EEC21B`; ערכות בהיר/כהה דרך CSS variables.
- טיפוגרפיה: גופן מערכת/Heebo-like, RTL, היררכיה נקייה.
- כרטיסים מעוגלים, צללים עדינים, אנימציות מעבר קצרות, מצבי hover/focus ברורים.
- לוגו `gav-yam.svg` ב-header ובשער ההדפסה.

## 10. הדפסה / PDF
- גיליון `@media print` ייעודי: מסתיר shell, פורש את כל הפרקים כ-A4, שער+לוגו, כותרת עליונה/תחתונה עם סיווג ומספרי עמודים. תוצאה קרובה ל-Word.

## 11. ייצוא `.docx` (שלב אחרון — ניתן לדחייה)
- שימוש ב-`docx` בצד-לקוח כדי לייצר את אותו מסמך ממותג מתוך ערכי האתר הנוכחי — גשר חזרה ל-Word.
- אם נדחה: כפתור מושבת עם תווית "בקרוב"; ה-PDF מכסה את הצורך.

## 13. שלבי בנייה (יפורטו בתוכנית)

3. מנוע רינדור: Kv / Table / Checklist / Bullets / Box / Callout.
4. store: state · autosave ל-localStorage · ריבוי אתרים · ייצוא/ייבוא JSON.

## 14. הגדרת "סיום" (Acceptance)

- מילוי נשמר אוטומטית ושורד רענון; ייצוא/ייבוא JSON עובד; ריבוי אתרים עובד.
- חיפוש, מחווני השלמה, מתג דוגמאות וולידציות פועלים.
- מצב כהה/בהיר; RTL תקין; רספונסיבי; נגיש בניווט מקלדת.
- הדפסה→PDF מפיקה מסמך A4 ממותג קריא.
- `npm run build` מייצר build סטטי שניתן לפתוח/לארח.

## 15. סיכונים / החלטות פתוחות
- **גופן עברי**: שימוש בגופן מערכת מול הטמעת Heebo (משקל מול נראות) — ייבחר בשלב 1.
- **ייצוא docx בדפדפן**: גודל bundle — נטען עצלן (lazy) רק בעת ייצוא.
- **התנגשות package.json**: האפליקציה ב-`app/` נפרדת לחלוטין מהשורש כדי לא לשבור את `generate.js`.
